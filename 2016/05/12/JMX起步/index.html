Java 管理扩展简单入门

说明

Java 管理扩展（Java Management Extension，JMX）
Sun 公司也在其 Java SE 5 版本中，正式提出了 Java 管理扩展（Java Management Extensions，JMX）用来管理检测 Java 程序（同时 JMX 也在 J2EE 1.4 中被发布）。
JMX 的提出，让 JDK 中开发自检测程序成为可能，也提供了大量轻量级的检测 JVM 和运行中对象 / 线程的方式，从而提高了 Java 语言自己的管理监测能力。

JMX起步

JMX （Management Extensions）是一个为应用程序，设备，系统等植入管理功能的框架。

JMX 规范可以分为三层：

  1. 设备层
    规范定义了编写可由 JMX 管理的资源的标准，即如何写 MBean
  2. 代理层
    定义了创建代理的规范，封装了 MBean Server
  3. 分布式服务层。
    主要定义了对代理层进行操作的管理接口和构件

一共有四种类型的MBean：

  1. 标准类型Mbean
  2. 动态类型 MBean
  3. 开放类型 MBean
  4. 模型类型 MBean

标准Mbean

基本的操作方法如下：

  1. 创建一个接口， 命名规范为： Java 类名 + MBean 后缀
  2. 实现上述的接口
  3. 创建代理，该代理包含 MBeanServer 实例
  4. 为 MBean 创建 ObjectName 实例，ObjectName 由两部分组成： domain 和 键/值对
  5. 实例化 MBean
  6. 将 MBean 注册到 MBeanServer
